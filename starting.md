## [00-05분] 파이썬과의 첫 만남 (환경 및 기본 개념)

### 1. 주피터 노트북의 특징 (Notebook vs Script)
*   **시연**: 셀에 `10 + 20`만 입력하고 실행해봅니다. 그 다음 `print(10 + 20)`을 실행해봅니다.
*   **설명**:
    > "보통의 프로그래밍 환경(.py 스크립트)에서는 `print` 명령어를 써야만 화면에 무언가가 나옵니다. 하지만 주피터 노트북은 여러분과의 대화를 위해 만들어졌습니다. 그래서 마지막 줄에 적힌 **표현식(Expression)**의 결과값을 자동으로 보여줍니다."
*   **주의점**: 셀 중간에 있는 값은 출력되지 않음을 보여줍니다. (예: `a=1` 엔터 `a` 엔터 `b=2`)

### 2. 표현식(Expression) vs 명령문(Statement)
*   **개념 정의** (칠판 판서 권장):
    *   **표현식(Expression)**: `3 + 5`, `"Hello"` 처럼 하나의 **값(Value)**으로 귀결되는 코드 덩어리입니다. 주피터는 이걸 보여줍니다.
    *   **명령문(Statement)**: `a = 10` 처럼 컴퓨터에게 "이걸 저장해", "반복해"라고 **명령**하는 문장입니다. 이건 그 자체로 값이 아니므로 화면에 아무것도 안 나올 수 있습니다.
*   **CS 포인트**: "프로그래밍 언어를 배울 때는 이 코드가 값을 만들어내는지(Evaluate), 아니면 상태를 변화시키는지(Execute) 구분하는 눈이 필요합니다."

## [05-15분] 자료형과 동적 타이핑 (Data Types)

### 1. "모든 것은 객체다"
*   **설명**: 파이썬 세상의 모든 데이터는 자신만의 '소속(Type)'이 있습니다. 숫자인 척 하는 문자(`"3"`)와 진짜 숫자(`3`)는 엄연히 다릅니다.
*   **실습 가이드**: `type()` 함수는 데이터를 만났을 때 인사하는 방법입니다. "너 정체가 뭐니?"
    *   `type(3)` -> `<class 'int'>`
    *   `type(3.14)` -> `<class 'float'>` (실수)
    *   `type("3")` -> `<class 'str'>` (문자열)

### 2. 동적 타이핑 (Dynamic Typing)
*   **시연**:
    ```python
    box = 100
    print(type(box))  # int
    box = "홍길동"
    print(type(box))  # str
    ```

## [15-25분] 변수, 메모리, 그리고 약속 (Variables)

### 1. 변수의 멘탈 모델 (Mental Model)
*   **오개념 수정**: "변수를 값을 저장하는 상자(Box)라고 생각하기 쉽지만, 파이썬에서는 **값(객체)을 가리키는 이름표(Reference)**라고 생각하는 것이 더 정확합니다."
*   **NameError 시연**: 정의하지 않은 변수 `my_score`를 출력하려 해봅니다.
    *   *메시지 해석*: "'my_score' is not defined" -> "컴퓨터 메모리에 'my_score'라고 붙어있는 포스트잇이 없는데요?"

### 2. Snake Case (뱀 표기법)
*   **설명**: 코드도 글입니다. 읽기 좋아야 합니다.
*   **비교**:
    *    (Camel Case) - 자바 스타일
    *    (Pascal Case) - 클래스 이름 스타일
    *    (**Snake Case**) - **파이썬 변수 스타일**
*   **강조**: "문법적으로 틀린 건 아니지만, 개발자들 사이의 약속(Convention)입니다. 혼자 튀는 코드를 작성하지 마세요."

## [25-30분] 연산자의 세계 (Operators)

### 1. 나눗셈의 세 가지 얼굴
*   가장 헷갈리기 쉬운 부분이므로 명확히 구분해줍니다.
    1.  `/` (**True Division**): 결과가 무조건 실수(`float`)입니다. `4 / 2`도 `2.0`이 됩니다.
    2.  `//` (**Floor Division**): 몫만 취하고 소수점 아래는 버립니다. 정수 나눗셈에 씁니다.
    3.  `%` (**Modulo**): 나머지를 구합니다.

### 2. 나머지 연산자(%)의 마법
*   **질문**: "나머지 연산은 초등학교 때 배우고 안 쓸 것 같지만, 컴퓨터 공학에서는 가장 중요한 연산자 중 하나입니다. 어디에 쓸까요?"
*   **활용 예시**:
    *   홀수/짝수 판별 (2로 나눈 나머지가 0인가 1인가)
    *   시간 계산 (초를 분/초로 변환할 때)
    *   암호학, 해시 테이블 등

### 3. 문자열 연산
*   **퀴즈**: "문자열끼리 더하면 어떻게 될까요? 곱하면요?"
*   **시연**:
    *   `"Py" + "thon"` -> `"Python"`
    *   `"Ha" * 3` -> `"HaHaHa"`
*   **의미**: 데이터 타입에 따라 연산자의 동작이 달라짐(**Overloading**)을 은연중에 보여줍니다.
